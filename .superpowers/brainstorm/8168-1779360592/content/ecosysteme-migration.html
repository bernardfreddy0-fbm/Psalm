Impact par composant

Composant Effort Ce qui change
PostgreSQL sur VPS ✅ Facile Template Coolify, pg_dump depuis Supabase, restore
Hono API (endpoints data) 🟡 Moyen ~20 endpoints (membres, planning, chants, events…)
Auth (JWT + emails) 🟡 Moyen better-auth ou Lucia, login/reset via Resend
PsalmAdmin frontend 🔴 Lourd Remplacer supabase/supabaseAdmin → fetch() dans api.ts (1 fichier principal)
PsalmMembre frontend 🟡 Moyen Même refactor api.ts, mais RLS remplacé par middleware Hono
Storage (partitions) ✅ Facile Minio sur VPS ou Cloudflare R2 (gratuit 10GB)
Module IA ✅ Propre S'intègre naturellement dans Hono — 0 Supabase
Estimation totale : 3–4 semaines en travaillant par phases.
La bonne nouvelle : tout est déjà sur OVH VPS. Pas de changement d'infrastructure, juste de stack applicative.
Résultat : zéro dépendance externe payante, zéro secret dans le frontend, un seul backend TypeScript pour tout.
