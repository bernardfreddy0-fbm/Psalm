Design — Migration Backend AEF (Spec 1)

5 phases progressives · Supabase reste actif jusqu'à validation complète

Phase 1 — Infrastructure (≈ 2 jours)

Objectif : Avoir le squelette Hono + PostgreSQL tournant sur Coolify, accessible en HTTPS.
• Créer service Coolify PostgreSQL 16 sur OVH VPS
• Importer le schéma Supabase via pg_dump --schema-only
• Créer service Coolify Node.js 22 (Docker) — projet Hono vide
• Configurer domaine api.a-e-f.fr avec SSL Let's Encrypt via Coolify
• Créer service Coolify Minio pour le stockage fichiers
• Variables d'environnement : DATABASE_URL, JWT_SECRET, RESEND_API_KEY, MINIO_*

Phase 2 — Auth (≈ 3 jours)

Objectif : Remplacer Supabase Auth par JWT Hono. C'est le bloc le plus critique — les deux apps en dépendent.
• Library : better-auth (TypeScript-first, supporte email/password + reset, bien maintenu)
• Endpoints : POST /auth/login · POST /auth/logout · POST /auth/refresh · POST /auth/reset-password · POST /auth/set-password
• JWT signé HMAC-SHA256, expiration 7j + refresh token 30j stocké en base
• Reset password via Resend (email avec lien tokenisé, valable 1h)
• Migrer les hash de mots de passe depuis Supabase (format bcrypt compatible)
• Tester en parallèle avant de couper Supabase Auth

Phase 3 — Endpoints data Hono (≈ 5 jours)

Objectif : Recréer tous les appels Supabase actuels en endpoints Hono typés.
• ORM : Drizzle (TypeScript-first, léger, SQL explicite, parfait avec PostgreSQL)
• Groupes d'endpoints : membres · planning · chants · événements · permissions · config · absences · disponibilités
• Middleware d'auth sur toutes les routes protégées (vérifie JWT Hono)
• Middleware de rôle pour les routes admin-only (remplace RLS Supabase)
• Upload partitions : Hono reçoit le fichier → stocke dans Minio → retourne URL publique
• Migration données : pg_dump --data-only depuis Supabase → restore PostgreSQL VPS

Phase 4 — Migration frontends (≈ 4 jours)

Objectif : Supprimer le SDK Supabase des deux apps, connecter sur Hono.
• PsalmAdmin : remplacer supabase.ts + supabaseAdmin → classe ApiClient avec JWT Bearer
• PsalmMembre : même refactor, la logique RLS migre vers les middlewares Hono
• Supprimer les dépendances @supabase/supabase-js des deux package.json
• Supprimer variables VITE_SUPABASE_* des configs Coolify
• Tests de non-régression sur toutes les pages des deux apps

Phase 5 — Extinction Supabase (≈ 1 jour)

Objectif : Couper Supabase proprement après validation complète en production.
• Vérifier en prod que les deux apps fonctionnent 100% sur Hono pendant 48h
• Exporter une dernière sauvegarde complète Supabase
• Désactiver le projet Supabase (plan Free supprimé automatiquement après inactivité)
• Archiver les variables d'environnement Supabase dans un coffre-fort sécurisé
